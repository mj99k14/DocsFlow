**상태:** 개발 중

---

## 1. 프로젝트 개요

### 문제 정의

기업에서 문서(계약서, 보고서, 기획서 등)가 업로드될 때 담당 부서를 사람이 직접 판단해서 전달해야 한다. 이 과정에서 다음 문제가 발생한다:

- 어느 부서로 보내야 할지 모르는 경우
- 잘못된 부서로 전달되어 처리 지연
- 문서 처리 현황을 한눈에 파악하기 어려움

### 해결책

PDF 문서를 업로드하면 Claude AI가 자동으로 내용을 분석하고 적합한 부서를 추천한다. 담당자가 최종 승인하면 해당 부서 Slack 채널로 알림이 전송된다.

## 3. 현재 구현된 기능 (v1)

### 3-1. 문서 업로드

- PDF 파일 업로드 (드래그앤드롭 / 클릭)
- 여러 파일 동시 업로드 가능
- PDF 파일만 허용 (확장자 검증)
- 중복 파일 추가 방지

### 3-2. AI 자동 분석

업로드 즉시 백그라운드에서 실행:

1. PDF 텍스트 추출
2. Claude API로 분석 (최대 3000자)
3. 결과 저장:
   - 문서 유형 (계약서 / 보고서 / 기획서 / 품의서 / 기타)
   - 추천 부서 (법무팀 / 재무팀 / 인사팀 / 경영기획팀)
   - 3줄 요약
   - 키워드
   - AI 신뢰도 (0.0 ~ 1.0)
   - 분류 근거

### 3-3. 대시보드

- 전체 문서 목록 (5초마다 자동 갱신)
- 통계 카드: 총 문서 수 / 처리 완료 / 분석 중 / 평균 정확도
- 파일명 검색
- 상태별 필터 (대기중 / 분석중 / 완료 / 승인됨 / 반려 / 보류)
- 부서별 필터
- 업로드 날짜 + 시간 표시 (Asia/Seoul)

### 3-4. 문서 상세

- AI 분석 결과 전체 확인 (요약, 키워드, 신뢰도, 근거)
- PDF 원본 다운로드
- 승인 처리 (Approve & Send to Slack)
  - 이미 처리된 문서는 버튼 비활성화

### 3-5. 승인 및 Slack 알림

- 승인 시 해당 부서 Slack 채널로 알림 전송
- 알림 내용: 파일명, 승인 부서, 승인자, 원본 파일 다운로드 링크
- 승인 이력 DB 저장

### 3-6. 설정

- 부서 목록 관리 (추가 / Slack 채널 수정)

## 4. 데이터 모델

```
documents          — 업로드된 문서 (파일명, 경로, 상태)
analysis_results   — AI 분석 결과 (문서유형, 요약, 키워드, 근거)
document_departments — AI 추천 부서 + 신뢰도
departments        — 부서 목록 + Slack 채널
approval_history   — 승인/반려/보류 이력
```

### 문서 상태 흐름

```
PENDING → ANALYZING → COMPLETED → APPROVED
                                 → REJECTED
                                 → HELD
              → FAILED
```

## 5. 미구현 / 향후 개발 (로드맵)

### 단기 (곧 구현)

| 기능 | 설명 |
|------|------|
| 부서 삭제 | 설정에서 부서 삭제 기능 |
| 반려/보류 버튼 | 현재 승인만 있음. DocumentDetail에 반려·보류 버튼 추가 |
| 10MB 파일 크기 제한 | UI에는 표시되어 있으나 실제 검증 미적용 |

### 중기

| 기능 | 설명 |
|------|------|
| 문서 삭제 | 대시보드에서 문서 삭제 |
| 알림 설정 실제 연동 | 설정 페이지 토글이 현재 UI만 존재 |
| Slack Webhook URL 관리 UI | 현재 코드에 하드코딩. 설정 페이지에서 관리 가능하게 |

### 장기

| 기능 | 설명 |
|------|------|
| Slack Bot Token 전환 | Incoming Webhook → Bot Token으로 교체해 채널 자동 생성 가능 |
| AI Agent 전환 | 단순 분류를 넘어 다단계 추론, 문서 간 관계 분석 |
| 로그인/인증 | 현재 인증 없음. 사용자별 권한 관리 |
| 이메일 알림 | Slack 외 이메일로도 알림 |

---

## 6. 현재 제약사항

- 로그인 없음 — 승인자가 항상 '관리자'로 고정
- Slack Webhook URL이 코드에 하드코딩 (`services/slack.py`)
- PDF 텍스트 추출 실패 시 분석 불가 (스캔본 이미지 PDF 미지원)
- 분류 가능한 부서가 4개로 고정 (법무팀, 재무팀, 인사팀, 경영기획팀)

## 7. API 엔드포인트 현황

| Method | Path | 설명 |
|--------|------|------|
| POST | `/documents/upload` | 문서 업로드 |
| GET | `/documents/` | 전체 문서 목록 |
| GET | `/documents/{id}` | 문서 상세 |
| GET | `/documents/{id}/file` | 원본 파일 다운로드 |
| GET | `/documents/{id}/history` | 승인 이력 |
| POST | `/documents/{id}/approve` | 문서 승인/반려/보류 |
| GET | `/departments/` | 부서 목록 |
| POST | `/departments/` | 부서 추가 |
| PUT | `/departments/{id}` | 부서 수정 |
